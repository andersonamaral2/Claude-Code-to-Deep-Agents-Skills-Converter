# Contributing

Thanks for your interest in contributing! Here's how you can help.

## How to contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix: `git checkout -b feature/my-feature`
3. **Make your changes** following the guidelines below
4. **Commit** with a clear message: `git commit -m "Add: new transformation for X"`
5. **Push** to your fork: `git push origin feature/my-feature`
6. **Open a Pull Request** against `main`

## Guidelines

- Keep both languages (English and Portuguese) in sync — if you update `SKILL.en.md`, update `SKILL.pt.md` too
- Follow the existing formatting and structure of the SKILL files
- Test your conversions with real Claude Code skills before submitting
- One feature or fix per PR

## Commit message convention

Use prefixes to categorize your commits:

- `Add:` new feature or transformation
- `Fix:` bug fix
- `Docs:` documentation changes only
- `Refactor:` restructuring without behavior change

## Reporting issues

- Use the issue templates provided
- Include the Claude Code skill (or a minimal example) that caused the problem
- Describe what you expected vs what happened

## Questions?

Open an issue with the **Question** label.
